# Event decision patterns

- **? nodes**: odds start event-heavy; each time you hit a non-event outcome, chances of **combat / shop / treasure** on later ? rooms **creep up**—long ? chains get less “free” over time.
- When an event has **3+** options, read each for immediate HP loss vs long-term payoff.
- Favor outcomes that fix a known hole (AoE, block, scaling) over raw gold if a hard fight is soon.
- Curses and max-HP trades: only if the deck carries the downside or upside solves a boss check.
- If HP is critical, prefer safe payouts and avoid gambling events unless potions/sustain exist.
